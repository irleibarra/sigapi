{{define "doc"}} API REST para SIGENU

Descripción de la API

Paginador

La ruta paginador recibe dos números que son el índice del primer elemento que se va a mostrar en la página, y la cantidad de elementos que tendrá la página. Por ejemplo /pagina/0/10 comenzará desde el elemento 0 mostrando una página de 10 elementos. La especificación formal de la ruta es {{.PagPath}}

Obtener registro según el identificador

La ruta sigenu-id devuelve el registro en la base de datos SIGENU asociado al identificador suministrado en la URL. Por ejemplo /sigenu-id/-6baf556d:13225553c27:-7fce devuelve el registro en SIGENU con ese identificador. La especificación formal de la ruta es {{.IdPath}}

Filtrar por nombre

La ruta nombre filtra todos los registros en SIGENU cuyo nombre coincide con el patrón suministrado. Por ejemplo, /nombre/juan devuelve todos los registros cuyo campo nombre contiene juan. La especificación formal de la ruta es {{.NamePath}}

Autenticar

La ruta /auth responde a un POST con el cuerpo {"user":"usuario", "pass":"contraseña"} una cadena (JSON Web Token) que sirve para identificar al usuario si las credenciales proporcionadas en el POST son válidas. Esta cadena caduca a la hora de emitida.

Evualuaciones

La ruta /eval responde a un GET con el header "authHd" asociado a la cadena enviada por /auth, con las evaluaciones del estudiante al que identifica esa cadena
